偷窃开源代码的 Github Copilot：除了有效，一无所有
======

![github-logo-2][1]

软件自由保护组织，一个提倡开源的非盈利性社群，在一个措辞严历的博客中宣布了退出 Github 平台。并号召它的支持者与成员们去谴责 Github。SFC 的博客断言，微软与 OpenAI 训练 AI 系统 Copilot 所使用的数据已可被在一个开源许可证中使用。开源代码不像一个捐助箱，无论何时何地何用都可使用。

这就像照相。即使摄像者并没有指控你去用他们的哪张照片，你仍然被要求介绍出处。但是，SFC 在它的[博客][2]中说，Copilot 在使用别人的代码片段时没有这么做：

"这与 GitHub 长期存在的问题相呼应，也是我们必须一同抛弃 Github 的原因。我们使用 Github 的托管服务时，几乎在所有的地方都为 Copilot 所尽力偷窥。Github 的问题实际上较他的替代品更大。我们不相信亚马逊、Atlassian、Gitlab 或者其他的盈利性的托管者是完美的。但，一个现实地比较是：Github 的“习性”远坏于这些替代品。"

Github 是世界开源软件实际上的仓库。这是 YouTube、Twitter 与 Reddit 的交际处，只为程序员与他们创造的代码。无疑，他是可替代的。
从一个软件仓库迁移到另一个，并不像用 Instagram 交换 TikTok。在 2018 年，微软用了接近七十亿美元来得到 Github。自那时起，在 Copilot 的开发上，微软一直在利用它 OpenAI 的主要赞助者的身份。并且，Copilot 只能通过微软的特别邀请或是付费订阅来被访问。因为微软和 OpenAI 在剥夺那些使用并信任 Copilot 代码的人的能力时，把别人的代码实际上揉杂为一体。所以 SFC 和其他开源提倡者都感到被凌辱。

Copilot 必须被扼杀。替代地，微软和 OpenAI 可以搞台时间机器，然后穿越回去在 Copilot 的数据库中标出每一个数据点。然后再允许他们创建一个新模型，一个可以给出完全可信的输出的模型。但是，利用人们的利益和狂野的西部一般的监管环境，总是比关心你所提供的产品和服务的道德规范更容易。
