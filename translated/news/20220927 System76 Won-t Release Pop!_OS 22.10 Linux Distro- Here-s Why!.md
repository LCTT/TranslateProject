System76 不会发布 Pop！_OS 22.10 Linux 发行版：原因如下！
======
不会有 Pop!_OS 22.10。因此，您无法使用 GNOME 43 .. 这就是原因！

![System76 Won't Release Pop!_OS 22.10 Linux Distro: Here's Why!][1]

Pop!_OS 是一个基于 Ubuntu 的发行版，它提供了出色的用户体验以及强大的软件套件。

Pop!_OS 版本通常值得等待他们最新的技术堆栈和他们为桌面环境体验带来的有用调整。

但是，您不会获得 Pop!_OS 22.10 版本。

System76希望专注于开发自己的基于 Rust 的 COSMIC 桌面环境，放弃 GNOME 以用于未来的版本。😲

如果您不知道，这里是我们在 COSMIC 桌面上为 Pop!_OS 提供的较早报道以提供清晰性：

哦，哇！Pop!_OS Linux 开发人员正在创建基于 Rust 的新桌面环境

### 没有pop！_OS 22.10 发布

新的 Pop!_OS 版本通常在 Ubuntu 的 LTS 和非 LTS 版本的新版本之后发布。

如果没有 Pop!_OS 22.10，一些用户可能会因为没有在 Pop!_OS 上获得 GNOME 43 体验而感到失望。

感谢OMG！Ubuntu！因为在Pop!_OS 开发人员Michael Murphy的Reddit 评论中发现了这一点。

迈克尔提到：

我们将把开发时间集中在用 Rust 实现 COSMIC 上，而不是 22.10。支持多个版本的 Ubuntu 需要付出很多努力，而且 6 个月的发布周期确实消耗了产品的开发时间和稳定性。

👏我相信这是一个绝妙的决定。去年，当他们宣布计划基于 Rust 的从头开始构建的桌面环境时，我一直想知道他们是如何实现的。
从头开始制作一些东西并达到已经在 Pop!_OS 上拥有基于 GNOME 桌面的出色体验的用户的期望是一项艰巨的任务。

### COSMIC 桌面环境

![pop os cosmic de early build][9]

看到 COSMIC 桌面环境的更多发展会很有趣。上次我们尝试（或看到模型）时，即使他们刚刚开始，它也激起了用户的兴奋。

从技术上讲，它自 2021 年以来一直在开发中，Pop!_OS 将于 2023 年发布。

它是按照 freedesktop互操作性规范用Rust编程语言编写的，并且完全消除了对 GNOME 及其 shell 扩展的任何依赖。

我们之前已经介绍了 COSMIC 桌面环境的早期预览版本，您可以在这里查看：

我试用了 System76 新的基于 Rust 的 COSMIC 桌面！

为什么这是一件好事？

Pop!_OS 的 LTS 版本已经收到比 Ubuntu 更多的定期包更新和内核升级，因此您无需担心无法获得 22.10 版本。

相反，我们应该期待 Pop!_OS 的下一次重大升级，他们真正地引入了 Rust-COSMIC 桌面环境。
